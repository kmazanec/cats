GitLab CI setup

Step 1. In the OpenEMR project on GitLab, go to Settings → CI/CD → Variables and add three variables (mark all Protected + Masked): 

CATS_WEBHOOK_URL    {{ webhook_url }}
CATS_WEBHOOK_SECRET {{ secret }}
CATS_PROJECT_ID     {{ project.id }}

Step 2. Add a post-deploy job to .gitlab-ci.yml on the OpenEMR side: 

notify-cats-regression:
  stage: .post
  needs: ["deploy"]
  rules:
    - if: $CI_COMMIT_BRANCH == "main"
  image: alpine:latest
  before_script:
    - apk add --no-cache curl python3
  script:
    - |
      BODY=$(printf '{"version_tag":"%s"}' "$CI_COMMIT_SHORT_SHA")
      SIG=$(printf '%s' "$BODY" | python3 -c "import sys,hmac,hashlib,os; \
        print('sha256=' + hmac.new(os.environ['CATS_WEBHOOK_SECRET'].encode(), \
        sys.stdin.buffer.read(), hashlib.sha256).hexdigest())")
      curl -fsS -X POST "$CATS_WEBHOOK_URL" \
        -H "Content-Type: application/json" \
        -H "X-CATS-Signature: $SIG" \
        --data-raw "$BODY"

The HMAC is computed over the exact bytes that are sent. Use --data-raw (not --data) so curl doesn't strip whitespace. Body is optional — an empty body verifies fine as long as the signature matches sha256("").
